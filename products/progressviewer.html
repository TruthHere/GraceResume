ProgressViewer 现场进度相册

Vibe Coding · Cursor · 微信小程序

微信小程序 PRD v1.2 体验版 1.0.0 现场管理

Why

施工现场需要定期拍照留痕，但照片堆在手机相册里，很快就对不上项目和位置。

Construction sites need regular photo documentation — but shots pile up in the phone gallery and quickly lose their project and location context.

在工装施工项目里，项目经理每周会到工地定点拍照，记录「某一层、某一处」建到了什么进度——这些照片是写周报、开会汇报、给业主或设计方同步情况的常用素材。现实中大家用手机随手拍，照片全部落在个人相册，和生活照、截图混在一起；照片本身也不会自动记下属于哪个项目、对应平面图上的哪个位置、哪一天拍的。过一两周再想找某次巡检的对比图，只能靠人工翻相册；临时通过微信转发几张，接收方也很难说清楚具体是哪儿。

On commercial fit-out projects, the project manager visits the site weekly to photograph fixed spots — documenting progress at “this floor, this location.” These photos feed weekly reports, status meetings, and updates to the client or design team. In practice, people shoot on their phones and everything lands in the personal gallery, mixed with daily life shots and screenshots. A photo doesn’t automatically record which project it belongs to, which spot on the floor plan it shows, or when it was taken. A week or two later, finding a specific inspection shot means scrolling and guessing; forwarding a few images on WeChat still leaves the recipient unsure where exactly they were taken.

What

在楼层平面图上点位置，拍照或选图归档，以后按日期和点位找照片。

Mark a spot on the floor plan, capture or pick a photo, archive it — then browse by date or location.

这是面向项目经理的微信小程序：创建项目、上传平面图，在图上选点或新建点位后，现场拍照或从相册选已有图，确认归档。照片绑定项目、平面图、点位与拍摄时间；项目相册支持「按日期」（默认）和「按点位」两种阅览；需要对外同步时，可一键生成带说明的分享图，通过微信发图。界面步骤少、按钮大，适合工地单手操作。

A WeChat mini program for project managers: create a project, upload floor plans, pick or add a point on the map, then shoot on-site or select an existing photo and confirm archive. Each photo is tied to project, plan, point, and capture time. The project album supports browsing by date (default) or by point. When sharing externally, generate a labeled summary image and send via WeChat. Few steps, large controls — built for one-handed use on site.

How

无编程基础，用 Cursor 独立从需求文档到可演示的体验版。

No coding background — independently shipped from PRD to a demo-ready beta with Cursor.

从 v1.0 需求骨架到 v1.2 完整闭环：先写 PRD 与流程，再在微信开发者工具中实现「选点 → 采源 → 确认归档 → 相册回看 → 分享图」主路径，并在真机上验收弱网、户外亮光与单手操作。技术选型为微信云开发：平面图存云端，照片存本机、业务元数据（项目、点位、时间等）存云数据库——在 PRD 测算下，典型工地用量月费可控制在基础套餐 19.9 元以内。

From v1.0 PRD skeleton to v1.2 full loop: wrote requirements and flows first, then built the main path — pick point → choose source → confirm archive → browse album → share image — in WeChat DevTools, with real-device testing for weak network, outdoor glare, and one-handed use. Stack: WeChat Cloud Base. Floor plans in cloud storage; photos on device; metadata (project, point, time) in cloud database. Per PRD estimates, typical site usage stays within the ¥19.9/month base plan.

已交付 PRD v1.2 ↔ 体验版 1.0.0，GitHub 存档；v1.3 规划相册读库优化与进度/事件记录模式，尚未发布新版体验包。

Delivered PRD v1.2 ↔ beta 1.0.0, archived on GitHub. v1.3 plans album read optimization and progress/event recording modes — no new beta uploaded yet.
