成本报告处理工具

Vibe Coding · Cursor · 纯前端 Web 工具

HTML / JavaScript 标准化 小试牛刀

Why

每个项目的成本报告、合同清单各有一套，但公司要汇总成统一数据库——中间那步「翻译」，现在全靠人工。

Every project’s cost report and contract breakdown looks different — yet the firm needs a unified database, and the “translation” step in between still runs on manual work.

在工作中我发现：各项目有自己的成本报告和合同清单模板，条目描述、科目划分、格式都不一致。但公司层面需要把这些数据汇总进统一模板，形成可统计、可对比的成本数据库——也就是说，要把每个项目的清单「翻译」成公司标准分类与层级。这个翻译过程过去靠 PM 或 QS 在 Excel 里逐行对照、手工归类，项目一多、条目一多，重复劳动量大，也容易因理解差异导致口径不统一。

On the job, I saw a recurring gap: each project carries its own cost report and contract breakdown — different wording, cost codes, and layouts. At company level, though, teams need to roll those up into a single template and build a comparable cost database. In practice, that means “translating” each project’s schedule into the firm’s standard breakdown. Today that translation is manual — PMs or QS staff line-by-line in Excel — slow at scale, and prone to inconsistent classification when different people apply the rules differently.

What

把项目清单按映射规则自动「翻译」到公司统一成本类别，复核后导出可直接入库的格式。

Auto-“translate” project line items into the company’s standard breakdown via mapping rules — review, then export in a database-ready format.

这是面向 PM / QS 的浏览器工具：可实现上传某项目的 .xlsx 成本清单，指定「合同内容」「合同金额」等列后，系统按预设的关键词映射规则，将项目条目自动匹配到公司统一模板类别。人工复核确认无误后，按公司模板要求设置导出格式，一键下载已「翻译」完成的 Excel，可直接用于汇总入库或进一步统计。

A browser tool for PMs and QS teams: upload a project’s .xlsx cost list, map columns for contract scope and amounts, then auto-match each line to the company’s standard major/minor categories (M / T structure) via configurable keyword mapping rules. Review row-by-row in the table — preview or edit mappings, refresh matching, override exceptions manually. When ready, apply the firm’s export formatting (bold, fill color, indent) and download a “translated” Excel ready for database roll-up or further analysis.

How

把工作中手工翻译的流程抽象成映射规则，用 Cursor 做成纯前端自动分类工具。

Abstracted the manual translation workflow into mapping rules — built a pure-frontend auto-classification tool with Cursor.

我先还原真实工作流：项目清单 → 对照公司模板逐条归类 → 调整格式 → 提交汇总。再把其中可重复的判断逻辑抽成关键词映射规则（如弱电、IT、综合布线等多种表述名称通常都指向 IT 相关成本），梳理「上传 → 选列 → 规则匹配 → 复核 → 导出」主路径，用 Cursor 实现页面与处理逻辑。映射规则可在界面内维护，同一套公司模板可反复用于不同项目的清单翻译，把原先的人工翻译转为规则驱动的自动分类。

I first mapped the real workflow: project schedule → line-by-line mapping to the company breakdown → format → submit for roll-up. Then I extracted the repeatable judgments into keyword mapping rules (“this description → that company code”) and built the main path — upload → map columns → rule matching → review → export — with Cursor. Pure frontend (SheetJS + FileSaver); mapping rules are editable in the UI. One company template can be reused across projects, turning manual translation into rule-driven auto-classification.
