One JLL 项目案例库

Vibe Coding · Cursor · HTML 交互原型

知识管理 投标 / BD 工装项目

Why

公司积累了很多项目案例，但投标或对标时，往往只有表格里一行摘要，找不到完整材料和可引用的成果。

The firm has years of project cases — yet when bidding or benchmarking, people often get only a one-line table summary, not the full materials or quotable outcomes.

在仲量联行做项目时，历史案例分散在 Excel 清单、个人文件夹和各类 PPT 里。列表里能看到客户名和一句话简介，但成果数据、客户反馈、完整案例 deck 常常对不上或缺失——BD、投标和技术标撰写时，很难快速回答「我们做过哪些类似项目、交付了什么、结果如何」。

At JLL, past cases lived across Excel lists, personal folders, and scattered decks. A row might show the client and a one-line blurb, but outcome data, client feedback, and the full case presentation were often missing or hard to match — slowing BD, bids, and technical proposals when teams needed to answer: “What similar work have we done, what did we deliver, and what were the results?”

What

把案例做成可检索的知识库：搜关键词、筛维度，从列表直达完整案例。

A searchable case library: keyword search, multi-filter narrowing, and a path from list to full case.

这是我在 JLL GPT 侧栏设计的「One JLL Case Library」交互原型。用户可按客户名、标签、摘要、服务内容等关键词检索，并用服务城市、业务线、客户行业、服务年份等条件组合收窄结果。列表展示客户、负责人、标签、摘要与服务内容；展开行可查看成果数据与客户反馈；进一步打开完整案例详情页，并跳转案例 PPT。

An interactive prototype for the JLL GPT sidebar: “One JLL Case Library.” Users search by client, tags, summary, or service scope, then narrow with city, business line, industry, and service year. The table shows client, lead, tags, summary, and services; expanding a row surfaces outcome metrics and client feedback; a detail view and PPT link complete the path from discovery to reusable experience.

How

从 PM 日常检索场景出发，用 Cursor 独立做出可演示的 HTML 原型。

Started from how PMs actually search cases — shipped a demo-ready HTML prototype with Cursor.

我先梳理业务人员需要的字段与筛选维度（如「搜 BIM 能立刻看到相关项目」「再选上海进一步收窄」），再实现列表、分页、行内展开与详情页等交互。演示视频完整走通：进入案例库 → 浏览全量列表 → 关键词检索 → 多维筛选 → 查看成果与反馈 → 打开完整案例。

I mapped the fields and filters PMs need (e.g. search “BIM” for related projects, then add “Shanghai” to narrow), then built list, pagination, row expansion, and detail views in HTML. The demo walks through: enter library → browse full list → keyword search → combined filters → outcomes and feedback → open full case.
