项目案例库

Vibe Coding · Cursor · HTML 交互原型

知识整合管理 公司级应用 快速验证

Why

公司积累了很多项目案例，但缺乏一个统一的平台整理宝贵的项目经验。

The firm has years of project cases — yet when bidding or benchmarking, people often get only a one-line table summary, not the full materials or quotable outcomes.

在公司做项目时，历史案例分散在 Excel 清单、个人文件夹和各类 PPT 里。部门并没有一个集各项目成果数据、过程经验的案例库。投标、技术标撰写或项目中解决新问题时，只有少数人能够快速回答「我们做过哪些类似项目、交付了什么、结果如何」。公司层面，相关业务线的案例也无法互通，各自为营，导致全生命周期业务的天然优势很难最大化。

On the job, past cases lived across Excel lists, personal folders, and scattered decks. A row might show the client and a one-line blurb, but outcome data, client feedback, and the full case presentation were often missing or hard to match — slowing BD, bids, and technical proposals when teams needed to answer: “What similar work have we done, what did we deliver, and what were the results?”

What

把案例做成可检索的知识库：搜关键词、筛维度，从列表直达完整案例。

A searchable case library: keyword search, multi-filter narrowing, and a path from list to full case.

因此我和几个同事一起设计了「项目案例库」，而我使用 Cursor 做出以下交互原型。用户可按客户名、标签、摘要、服务内容等关键词检索，并用服务城市、业务线、客户行业、服务年份等条件组合收窄结果。列表展示客户、负责人、标签、摘要与服务内容；展开行可查看成果数据与客户反馈；进一步打开完整案例详情页，并跳转案例 PPT。

My colleagues and I designed a case library concept, and I built the interactive prototype below with Cursor. Users search by client, tags, summary, or service scope, then narrow with city, business line, industry, and service year. The table shows client, lead, tags, summary, and services; expanding a row surfaces outcome metrics and client feedback; a detail view and PPT link complete the path from discovery to reusable experience.

How

从 PM 日常检索场景出发，用 Cursor 独立做出可演示的 HTML 原型。

Started from how PMs actually search cases — shipped a demo-ready HTML prototype with Cursor.

我先梳理业务人员重点关注的字段与筛选维度，再实现列表、分页、行内展开与详情页等交互。演示视频完整走通：进入案例库 → 浏览全量列表 → 关键词检索 → 多维筛选 → 查看成果与反馈 → 打开完整案例。

I mapped the fields and filters PMs need (e.g. search “BIM” for related projects, then add “Shanghai” to narrow), then built list, pagination, row expansion, and detail views in HTML. The demo walks through: enter library → browse full list → keyword search → combined filters → outcomes and feedback → open full case.
